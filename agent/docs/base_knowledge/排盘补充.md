## 一、年上起月法（五虎遁）

月的地支是固定的（正月寅、二月卯...），但天干需要根据年干推算。

口诀：

> 甲己之年丙作首，乙庚之岁戊为头，
> 丙辛必定寻庚起，丁壬壬位顺行流，
> 若问戊癸何处起，甲寅之上好追求。

### 算法

根据年天干查表得正月（寅月）的天干，然后逐月顺推：

| 年天干 | 正月（寅） | 二月（卯） | 三月（辰） | ... | 十二月（丑） |
|--------|-----------|-----------|-----------|-----|-------------|
| 甲、己 | 丙寅 | 丁卯 | 戊辰 | ... | 丁丑 |
| 乙、庚 | 戊寅 | 己卯 | 庚辰 | ... | 己丑 |
| 丙、辛 | 庚寅 | 辛卯 | 壬辰 | ... | 辛丑 |
| 丁、壬 | 壬寅 | 癸卯 | 甲辰 | ... | 癸丑 |
| 戊、癸 | 甲寅 | 乙卯 | 丙辰 | ... | 乙丑 |

### Python实现

```python
# 年干 → 正月天干
YEAR_STEM_TO_MONTH_STEM = {
    '甲': '丙', '己': '丙',
    '乙': '戊', '庚': '戊',
    '丙': '庚', '辛': '庚',
    '丁': '壬', '壬': '壬',
    '戊': '甲', '癸': '甲',
}

STEMS = ['甲', '乙', '丙', '丁', '戊', '己', '庚', '辛', '壬', '癸']
BRANCHES = ['子', '丑', '寅', '卯', '辰', '巳', '午', '未', '申', '酉', '戌', '亥']

def get_month_stem_branch(year_stem, month_branch_index):
    """
    year_stem: 年天干字符
    month_branch_index: 月地支序号 (寅=0, 卯=1, ..., 丑=11)
    返回: (月天干, 月地支) 字符串
    """
    start_stem = YEAR_STEM_TO_MONTH_STEM[year_stem]
    start_idx = STEMS.index(start_stem)
    stem_idx = (start_idx + month_branch_index) % 10
    return STEMS[stem_idx] + BRANCHES[(2 + month_branch_index) % 12]
    # 注：寅在BRANCHES中index=2
```

### 注意：以节气为月界

月份不以农历初一为界，而是以节气：

| 月支 | 节气起始 |
|------|---------|
| 寅 | 立春 |
| 卯 | 惊蛰 |
| 辰 | 清明 |
| 巳 | 立夏 |
| 午 | 芒种 |
| 未 | 小暑 |
| 申 | 立秋 |
| 酉 | 白露 |
| 戌 | 寒露 |
| 亥 | 立冬 |
| 子 | 大雪 |
| 丑 | 小寒 |

## 二、日上起时法（五鼠遁）

时辰的地支是固定的（23-1点子时、1-3点丑时...），天干需要根据日干推算。

口诀：

> 甲己还加甲，乙庚丙作初，
> 丙辛从戊起，丁壬庚子居，
> 戊癸何方发，壬子是真途。

### 算法

根据日天干查表得子时的天干：

| 日天干 | 子时 | 丑时 | 寅时 | ... | 亥时 |
|--------|------|------|------|-----|------|
| 甲、己 | 甲子 | 乙丑 | 丙寅 | ... | 乙亥 |
| 乙、庚 | 丙子 | 丁丑 | 戊寅 | ... | 丁亥 |
| 丙、辛 | 戊子 | 己丑 | 庚寅 | ... | 己亥 |
| 丁、壬 | 庚子 | 辛丑 | 壬寅 | ... | 辛亥 |
| 戊、癸 | 壬子 | 癸丑 | 甲寅 | ... | 癸亥 |

### Python实现

```python
DAY_STEM_TO_HOUR_STEM = {
    '甲': '甲', '己': '甲',
    '乙': '丙', '庚': '丙',
    '丙': '戊', '辛': '戊',
    '丁': '庚', '壬': '庚',
    '戊': '壬', '癸': '壬',
}

# 时辰地支固定映射
HOUR_TO_BRANCH = {
    0: '子', 1: '子',    # 23-1点
    2: '丑', 3: '丑',    # 1-3点
    4: '寅', 5: '寅',    # 3-5点
    6: '卯', 7: '卯',    # 5-7点
    8: '辰', 9: '辰',    # 7-9点
    10: '巳', 11: '巳',  # 9-11点
    12: '午', 13: '午',  # 11-13点
    14: '未', 15: '未',  # 13-15点
    16: '申', 17: '申',  # 15-17点
    18: '酉', 19: '酉',  # 17-19点
    20: '戌', 21: '戌',  # 19-21点
    22: '亥', 23: '亥',  # 21-23点
}

def get_hour_stem_branch(day_stem, hour):
    """
    day_stem: 日天干字符
    hour: 0-23的钟点
    返回: (时天干, 时地支) 字符串
    """
    branch = HOUR_TO_BRANCH[hour]
    start_stem = DAY_STEM_TO_HOUR_STEM[day_stem]
    branch_idx = BRANCHES.index(branch)
    start_idx = STEMS.index(start_stem)
    # 从子时到当前时辰的偏移
    offset = branch_idx  # 子=0
    stem_idx = (start_idx + offset) % 10
    return STEMS[stem_idx] + branch
```

## 三、公历日期 → 日干支推算

这是排盘中最关键也最复杂的算法。以下给出标准数学方法。

### 方法：基于已知基准日的推算

以 **1900年1月1日 = 甲戌日** 为基准（该日是公认的甲戌日）。

```python
from datetime import date

# 1900-01-01 = 甲戌 (天干甲=0, 地支戌=10)
BASE_DATE = date(1900, 1, 1)
BASE_STEM = 0   # 甲
BASE_BRANCH = 10  # 戌

def date_to_ganzhi(d):
    """公历日期 → (天干, 地支)"""
    delta = (d - BASE_DATE).days
    stem = (BASE_STEM + delta) % 10
    branch = (BASE_BRANCH + delta) % 12
    return (STEMS[stem], BRANCHES[branch])
```

更高效的做法——年基数法（已知某年1月1日干支，推算全年）：

```python

def date_to_ganzhi_fast(d):
    """快速日干支推算（2000-2099年）"""
    year = d.year
    base = YUANDAN_GANZHI[year]
    day_of_year = d.timetuple().tm_yday - 1  # 0-indexed
    gz_index = (base + day_of_year) % 60
    stem = gz_index % 10
    branch = gz_index % 12
    return (STEMS[stem], BRANCHES[branch])
```

## 四、旬空（空亡）计算

### 原理

六十甲子分为六旬，每旬10天。天干10个为一轮，地支12个，每旬必有两个地支轮空：

| 旬 | 范围 | 空亡 |
|----|------|------|
| 甲子旬 | 甲子→癸酉 | 戌亥 |
| 甲戌旬 | 甲戌→癸未 | 申酉 |
| 甲申旬 | 甲申→癸巳 | 午未 |
| 甲午旬 | 甲午→癸卯 | 辰巳 |
| 甲辰旬 | 甲辰→癸丑 | 寅卯 |
| 甲寅旬 | 甲寅→癸亥 | 子丑 |

### 算法

给定日干支（天干tg, 地支dz），其中天干序号 甲=1...癸=10，地支序号 子=1...亥=12：

```python
def get_xunkong(day_stem, day_branch):
    """
    根据日干支计算旬空的两个地支
    day_stem: 天干字符
    day_branch: 地支字符
    返回: (空亡地支1, 空亡地支2)
    """
    tg = STEMS.index(day_stem) + 1   # 1-10
    dz = BRANCHES.index(day_branch) + 1  # 1-12

    # 第一步：求旬首地支
    # 旬首 = 当前地支 - (当前天干 - 1)
    xunshou = dz - (tg - 1)
    if xunshou <= 0:
        xunshou += 12

    # 第二步：空亡 = 旬首往前两位（旬首+10, 旬首+11，模12）
    kong1 = (xunshou + 10 - 1) % 12 + 1  # 地支序号
    kong2 = (xunshou + 11 - 1) % 12 + 1

    return (BRANCHES[kong1 - 1], BRANCHES[kong2 - 1])

# 验证：
# get_xunkong('甲', '子') → ('戌', '亥')  ✓
# get_xunkong('丙', '寅') → ('戌', '亥')  ✓ (同在甲子旬)
# get_xunkong('丙', '子') → ('申', '酉')  ✓ (在甲戌旬)
# get_xunkong('戊', '申') → ('午', '未')  ✓ (在甲申旬)
```

### realzhangliu 的实现

realzhangliu 使用了一种更巧妙的数学方法（liuyao.py 第85-93行）：

```python
# 地支配数: 子1...亥12
# 天干配数: 甲1...癸10
# cha = 地支数 - 天干数
# 如果 cha < 0: cha += 10
# 空亡 = 地支序列[cha-2], 地支序列[cha-1]
```

这个公式利用了Python的负数索引——当cha=0时取-2、-1即戌、亥，恰好是甲子旬的空亡。数学本质上等价于上面给出的标准公式。

## 五、寻伏神

当占得的卦中缺少某个六亲（比如占求财但卦中无妻财爻），需要从本宫首卦（八纯卦）中找出缺的那个六亲，即为 **伏神**。伏神所在的爻位称为 **飞神**。

### 算法

```python
def find_fushen(gua_yao_list, gong_name, needed_qin):
    """
    在本卦中寻找缺失六亲的伏神
    
    gua_yao_list: 本卦6个爻的六亲列表，如 ['父戌土', '兄申金', '官午火', ...]
    gong_name: 本卦所属宫名，如 '乾宫'
    needed_qin: 需要的六亲，如 '财'（妻财）
    
    返回: (伏神六亲纳甲字符串, 飞神所在爻位1-6) 或 None
    """
    # 先检查是否真的缺失
    existing_qins = {yao[:2] for yao in gua_yao_list}  # 取六亲前两字如'父','兄','官','财','孙'
    if needed_qin in existing_qins:
        return None  # 不缺失，不需要寻伏神
    
    # 获取本宫首卦（八纯卦）的六爻数据
    gong_shou_gua = GONG_SHOU_GUA[gong_name]  # 如乾宫的乾为天卦
    
    # 在本宫首卦中找needed_qin
    for i, yao in enumerate(gong_shou_gua):
        if yao.startswith(needed_qin):
            # 检查本卦该爻位是否有六亲（该位不应有同五行六亲）
            fushen = yao
            feishen_wei = i + 1  # 爻位 1-6
            return (fushen, feishen_wei)
    
    return None  # 理论上不会发生
```

### 本宫首卦数据

需要存储八纯卦的六爻六亲纳甲数据（参见第九节六十四卦查找表，每宫的第一卦即本宫首卦）。

### 示例

占得 **天地否**（乾宫），求财需看妻财爻：
- 天地否六亲：父戌土、兄申金、官午火、财卯木、官巳火、父未土
- 有「财卯木」在三爻 → 不缺失，无需寻伏神

占得 **天风姤**（乾宫），求财：
- 天风姤六亲：父戌土、兄申金、官午火、兄酉金、孙亥水、父丑土
- 无妻财爻！需从乾宫首卦（乾为天）找：乾为天中「财寅木」在二爻
- 伏神：妻财寅木；飞神：二爻的子孙亥水

## 六、装卦身（月卦身）

部分断卦体系中，除了世爻，还需确定 **卦身**（月卦身）。卦身也是断卦的重要参考。

### 规则

以世爻的地支来确定卦身：

> 阴世则从午月起，阳世还从子月生。
> 欲得识其卦中意，从初数至世方真。

- **世爻为阳爻**（─）：从初爻起子，顺数至世爻，看世爻落何地支，该地支即为卦身所在
- **世爻为阴爻**（- -）：从初爻起午，顺数至世爻，看世爻落何地支，该地支即为卦身所在

### 简化算法

其实有一个更简单的规律——看世爻的地支：
- 卦身地支 = 世爻地支。然后在全卦六爻中找相同的那个地支，该爻就是卦身。

但注意卦身不一定出现在本卦中，若不出现则卦无卦身。

## 八、完整排盘流程

综合以上知识，从零开始排盘的完整流程如下：

```
输入: 摇卦结果（6个爻，每爻为少阳/少阴/老阳/老阴）
       或 时间/数字/汉字起卦参数

1. 起卦
   - 确定6个爻的阴阳和动爻（老阳老阴为动爻）
   - 得到本卦（阳爻=1，阴爻=0 构成的6位编码）
   - 得到变卦（动爻阴阳反转）

2. 确定四柱
   - 年月日时 → 四个天干地支柱
   - 月界以节气为准

3. 查宫定卦
   - 用本卦编码查64卦查找表
   - 获取：所属宫、五行属性、世爻位、应爻位、六冲/六合、游魂/归魂

4. 装纳甲（干支）
   - 查表获取每爻的天干地支（已在查找表的六亲条目中体现）

5. 装六亲
   - 查表获取每爻的六亲（已在查找表中）

6. 装六兽
   - 按日干确定初爻配何六兽，逐爻顺排

7. 装变爻
   - 动爻的变卦六亲查表获取
   - 注意：变爻六亲仍以本卦宫五行属性为准

8. 寻伏神（如有缺失六亲）
   - 从本宫首卦中找缺失的六亲

9. 计算旬空
   - 按日干支计算空亡的两个地支

10. 装卦身（可选）
    - 按世爻位确定卦身

输出: 完整的排盘结果
```

### 编码说明

6位字符串中，1=阳爻（─），0=阴爻（- -）。

老阳(○)和老阴(×)在起卦时记录，确定动爻后转为对应的阳(1)阴(0)来编码查表。

每个六亲条目格式为 `六亲+地支+五行`，如 `父戌土` = 父母爻 + 戌 + 土。
