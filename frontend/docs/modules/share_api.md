# Share API Contract (UI Simulation)

Last updated: 2026-04-16 (v0.6.4)

## Scope

Interpretation card render/edit/share pipeline.
Visual card canvas supports theme selection, custom background images, and text editing.

## Auth & Security

- Render endpoints can be guest.
- Save/publish endpoints require auth: `Authorization: Bearer <token>`
- External share endpoints should return share payload; client triggers native SDK.

## Data Models (JSON)

### `ShareCardDraft`

```json
{
  "cardId": "card_abc",
  "theme": "warm|cool|dark",
  "text": "string",
  "backgroundImageUrl": "string|null"
}
```

## Backend Anchors

- `share_api#card-render`
- `share_api#external-share`
- `share_api#card-theme`: client renders card via `ShareCardCanvas` with `ShareCardTheme` enum; backend may provide server-rendered cards for consistency
- `share_api#card-capture`: client uses `RepaintBoundary` to capture card as image; backend should provide high-res render endpoint for production quality
- `share_api#save-image`: persist generated PNG to device gallery with permission handling and failure states

## Endpoints (Reserved)

- `POST /share/card/render` — generate card image server-side with theme + background
- `POST /share/card/save` — save card draft
- `POST /share/community/publish` — publish card to community feed
- `POST /share/external` — trigger external share (WeChat/moments)
- `POST /share/card/render-with-theme` — render card with specific theme and optional custom background image URL

## Endpoint Details

- `POST /share/card/render`
  - Request: `{ "cardId": "card_abc", "theme": "warm", "backgroundImageUrl": null, "text": "string" }`
  - Response: `{ "success": true, "data": { "imageUrl": "https://cdn/.../card.png" } }`
- `POST /share/card/save` (idempotent)
- `POST /share/community/publish` -> should create a community post and return `postId`
- `POST /share/external` -> return structured payload for WeChat/moments

## Error Codes

- `40001` invalid payload
- `40101` auth required
- `40401` card not found
- `50000` internal error

## UI Component Notes

- `ShareCardCanvas`: 3:4 aspect ratio, renders hexagram lines via CustomPaint, coordinate name, summary text, footer
- `ShareCardWorkflowScreen`: live preview + theme row + background picker + text editor + three action buttons
- Export buttons now simulate capture success with generated byte count before showing success feedback
- `ImagePicker` used for custom background; backend should accept image upload and return CDN URL

## Replacement Notes

- Current behavior is UI-only simulation for card generation/edit/share actions.
- External share actions should later be connected with native SDK callbacks (WeChat SDK, system share sheet).
- Save/share actions already complete card capture locally but do not yet call native gallery/share plugins.
- Publish action should return a canonical post id so message/profile linkage can be rebuilt from backend events.
- Card capture currently uses client-side `RepaintBoundary`; production may prefer server-rendered images for consistent quality across devices.
